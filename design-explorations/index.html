2 directions · graph + detail

Two surviving directions for a GitHub-style issue site. Each is a self-contained mockup wired to the same demo data: switch projects, browse initiatives and ADRs, drill into an initiative, and now open any PRD, ADR, user story or issue for its full detail, plus a relationship graph at both the project and initiative level.

New in this round: a 🕸️ Graph view (project-wide and per-initiative) where every node is clickable, and expandable detail panels for PRDs (summary + goals + linked stories), ADRs (context / decision / consequences), user stories (narrative + acceptance criteria + implementing issues), and issues (description + tasks + parent story). Cross-links navigate between records.
